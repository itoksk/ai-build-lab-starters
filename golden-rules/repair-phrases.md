# Repair & polish phrases

Magic phrases you can copy as-is. The repair ones are your safety net when something breaks; the polish ones push a "60%" build toward done. Verbatim from the Skill pages.

> Live: https://ai-build-lab.pages.dev/skills/repair-without-breaking · /skills/error-paste-verbatim · /skills/seven-golden-rules

## Repair (don't break what works)

The repair magic phrase — **paste the full error after the colon**:

```
Don't break the existing code structure, keep all comments and logs, and fix only the related part. The problem: 
```

When it still won't fix — the escape route:

```
go back to the last working version
```

### How to report the error

- Copy the **entire** red error message (including the stack trace), not a snippet.
- Don't summarize it in your own words — "it's broken" loses the information that fixes it.
- Paste the full error right after the repair phrase above.

Pros paste the full error; beginners describe it in their own words — that's the divide.

## Polish (60% → 100%)

```
Take this from 60% to 100%.
```

```
Make it look like a site that would be featured on awwwards.com.
```

```
Make it feel smooth and fluid, with subtle animations.
```

```
Make the design not look AI-generated. Polish the details: icons, spacing, wording.
```

```
Make the UI clean and refined like Stripe's checkout.
```

```
Make it overwhelming in quality. Go beyond the limits.
```
